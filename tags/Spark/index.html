Spark#2 RDD编程(上)

RDD编程的概念

¶概述

每个Spark程序都包含一个运行main()方法的驱动程序，这个驱动程序能够在集群多个节点上并行操作。Spark提供了一种叫做resilient distribute dataset(RDD)的抽象，这个结构是在节点上运行程序的数据集合(程序)。RDD能由Hadoop文件系统中的文件初始化得到，也可以由驱动程序中的Scala集合转化得到。用户可以将RDD集合保存在内存中，这样可以让Spark程序高效的重用多个并行程序，减少以前Hadoop处理复杂业务出现的问题(HadoopMap-Reduce过程后会将结果存在硬盘，导致如果需要多个Reduce过程的情景时出现大量磁盘IO，性能下降严重)。RDD也能将数据从故障节点中恢复。

Spark还能让并行程序之间共享变量。默认情况下，Spark在集群中的不同节点上运行一个function来执行一组任务，它会向所有节点发送一个变量副本。如果一个变量需要被多个任务或者任务与驱动程序之间共享，Spark支持两种方式共享变量：broadcast variables(广播变量，在每个节点的内存中缓存一个值)和accumulators(累加器，仅仅允许变量执行加法，比如计数器和算术和)

本章将会演示Scala完成这些功能

Spark#4 Spark SQL概述

Spark SQL概述

Spark SQL是用于结构化数据处理的Spark模块。与Spark RDD API不同，Spark SQL提供的接口可以为Spark提供更多的数据及其计算时的结构信息。Spark使用这些额外的信息实现一些优化。用户可以使用SQL或者Dataset API对SparkSQL进行操作。由于它和用户使用的API或者开发语言无关，这意味着开发人员可以在多个API中轻松地切换
